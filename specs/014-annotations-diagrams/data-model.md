# Data Model: Annotations and Diagram Authoring

## Annotation

Fields:

- `id`: Stable text identifier generated by `spine/`.
- `target_kind`: One of `capture`, `local_file`, `working`, `archive`.
- `target_id`: Text id of the annotated target within its kind.
- `selection_start`: Optional integer character offset in the target text.
- `selection_end`: Optional integer character offset in the target text.
- `selection_text`: Optional text snapshot of the selected passage.
- `comment`: Non-empty user-authored annotation text.
- `created_at`: ISO timestamp set on create.
- `updated_at`: ISO timestamp set on create and changed on future edits if edit support is added.

Validation rules:

- `target_kind`, `target_id`, and non-empty trimmed `comment` are required.
- `target_kind` must be one of the four supported values.
- If both offsets are present, `selection_start` must be greater than or equal to `0`, and `selection_end` must be greater than `selection_start`.
- Empty or whitespace-only selections cannot create a passage annotation.
- Source documents are never modified by annotation create/delete.

Relationships:

- Belongs to exactly one Annotation Target.
- Produces one local QMD annotation index document while retained.

Lifecycle:

- Created from a Surface selection and comment.
- Listed for a target when the target is opened.
- Deleted by user request; corresponding QMD annotation index document is removed or refreshed so deleted text stops matching search.

## Annotation Target

Fields:

- `kind`: One of `capture`, `local_file`, `working`, `archive`.
- `id`: Existing source id for the target kind.
- `text`: Current readable text when available.

Validation rules:

- Target identity must be present for create/list/delete context.
- If current text is unavailable, saved annotation records remain intact and can still expose their saved selected text/comment.

## Diagram Block

Fields:

- `source`: Mermaid text inside a fenced markdown block.
- `markdown_range`: Position in the working document content, managed by the editor document text.
- `render_state`: Derived UI state: valid render, pending render, or non-destructive render error.

Validation rules:

- Diagram blocks are only authored in working docs.
- Invalid Mermaid syntax must not delete or rewrite the source text.
- Diagram source remains part of working-doc markdown and therefore searchable through existing working-doc indexing.

## Search Result Context

Fields:

- `kind`: Existing result kind plus annotation-aware context.
- `target_kind`: Present when the match came from annotation text.
- `target_id`: Present when the match came from annotation text.
- `annotation_id`: Present when the match came from annotation text.
- `snippet`: Source, annotation, or diagram-source snippet explaining why the result matched.

Validation rules:

- Annotation matches must lead back to the annotated source target.
- Deleted annotations must not produce search result matches.
